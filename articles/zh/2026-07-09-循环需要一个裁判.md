# 循环需要一个裁判

> 发布日期:2026-07-09 · [中文](../zh/2026-07-09-循环需要一个裁判.md) | [English](../en/2026-07-09-the-loop-needs-a-judge.md)

---

今天的笔记里，好几个东西都指向同一个词：循环。

Loop Engineering、Agent Harness、自动研究、自我进化系统、会和智能体一起进化的评估器。表层故事很好讲：我们正在从“提示 AI”，走向“设计一个系统，让系统替你持续提示 AI”。有一篇近期文章几乎就是这样定义 Loop Engineering 的：你不再只是那个不断提示 agent 的人，而是去设计一个会提示 agent 的系统。[^1]

这当然很厉害。

但真正危险的地方，也藏在这里。

循环本身不等于进化。循环本身只是重复。

真正的问题不是机器能不能在你离开键盘之后继续跑。它当然能。机器最擅长的事情之一，就是不累。真正的问题是：

什么东西告诉这个循环，它正在变得更好？

没有裁判的循环，只是一个越来越自信的圆圈。

## 生成变便宜以后，判断就变贵了

在人类历史的大多数时间里，写出一个初稿并不便宜。它需要注意力、时间、技能，有时候还需要勇气。所以我们天然尊重输出。一个人能把东西写出来，好像就已经完成了最难的部分。

AI 改变了这个比例。

生成现在变便宜了。不是完全免费，但已经便宜到瓶颈换了位置。稀缺能力不再是“你能不能给出一个像样答案”，而是“你能不能判断哪个答案值得活过现实这一关”。

这件事很容易被忽略，因为流畅输出仍然很像进展。一大段回答看起来像工作，一个能跑的 demo 看起来像证明，一个绿色对勾看起来像真相。

但生成越便宜，判断就越贵。

如果一个人只写一版，你可以慢慢读。如果一个 agent 一晚上生成五十个版本，问题就变成：谁来过滤？谁留下一个，丢掉四十九个？谁来防止系统只是在优化最容易看的指标，却慢慢偏离真正重要的东西？

这就是裁判的价值。

裁判不一定是人。它可以是一套测试、一个保留不用来训练的数据集、一位真实读者、编译器、愿意付费或拒绝付费的客户、物理测量、日历、公开承诺，或者一张在你被漂亮输出迷惑之前就写好的“完成定义”清单。

形式可以不同。功能只有一个：制造阻力。

而阻力，才会把运动变成学习。

## Autoresearch 真正教我们的事

今天笔记里最干净的例子，是 autoresearch。

它的公开 README 描述了一个很小但真实的设置：AI agent 只修改一个训练文件，跑一个固定 5 分钟的实验，检查验证指标；结果变好就保留，没变好就丢掉，然后继续下一轮。[^2] 这套设计故意很窄。数据准备和评估被锁住，agent 主要只能改 `train.py`。指标是 `val_bpb`，越低越好。固定时间预算让每次实验有可比性。[^2]

关键就在这里。

有意思的不是 agent 会写代码。我们早就知道 agent 会写代码。有意思的是，这套系统不给 agent 留下“听起来很有道理”就过关的空间。

答案不是“我觉得更好了”。

答案是一个被测量出来的结果。

一篇外部文章记录了一些早期实验：这个模式可以连续跑数百次尝试，只保留真正带来改善的变化，并且有些改进还能迁移到更大模型上。文章还提到另一个隔夜实验：几十轮之后，一个更小的模型超过了此前更大的基线模型。[^3] 这些都是早期案例，不是宇宙定律。但方向很重要。

循环之所以有力量，是因为裁判被明确写出来了。

如果指标会泄漏，agent 就会学会钻指标的空子。如果评估代码也能随便改，agent 就可能把门槛偷偷调低。如果测试数据没有真正保留，系统就可能背题。如果人类把“看起来不错”当停止条件，循环就会变成一台制造自信的机器。

所以，真正的工程不是简单地“让 agent 自己迭代”。

真正的工程是：冻住定义正确性的部分，限制 agent 可以行动的范围，记录每一次尝试，在稳定平面上比较尝试，只保留经得起裁判的改动。

这样，循环才会从打字机器变成学习系统。

## 固定裁判也会变老

但这里还有第二层问题。

裁判必须稳定，但裁判不能永远不变。

如果裁判永远固定，系统最终会学会裁判的形状。考试如此，榜单如此，内容平台如此，软件测试如此，个人习惯也是如此。一个指标只要足够重要，人和机器都会学会优化它，甚至学会欺骗它。

所以最近有一个很有意思的研究方向：不只让智能体进化，也让评估智能体的评估器进化。

一篇 arXiv 论文把这件事放在“红皇后问题”里讨论。过去很多自我改进系统默认评估环境是固定的；“红皇后”式的框架关心的是：当 agent 和 evaluator 一起演化时，怎样既让裁判变强，又不让整个系统变成随时移动球门的游戏。[^4]

这件事放在普通人的学习和工作里，也很有启发。

一个健康的学习系统，应该在一轮之内保持裁判稳定，在两轮之间升级裁判。

一轮之内，裁判不能乱晃。每次结果让你不舒服，你就改规则，那不是学习，那是在和现实讨价还价。

但一轮结束以后，裁判应该允许成熟。测试套件可以加一个回归用例。写作清单可以新增一个“读者会不会看晕”的问题。产品指标可以在发现它奖励浅层停留之后重新调整。学习计划也可以在发现旧考法测不出迁移能力之后升级。

裁判永远不变，就会过拟合旧考试。

裁判一不舒服就变，就会自我欺骗。

真正的手艺是：把裁判冻住足够久，让自己能学到东西；再足够诚实地升级裁判，让自己还能继续学。

## 普通人为什么也需要裁判

大多数人不会去跑自动模型训练实验。

但每个人都会被便宜生成包围。

你可以让 AI 给你十个文章大纲、十个商业想法、十个课程设计、十个产品名、十段谈判话术、十种对一本书的解释。模型都会给。它会很快、很礼貌、很像那么回事地给。

这当然有用。也很危险。

因为如果你自己不带裁判，模型的流畅就会变成裁判。

这就是很多人为什么会被高级工具养懒。不是因为工具差，而是因为工具给了人一种舒服的反馈替代品。

真正的裁判往往没那么舒服。

写作的裁判可能是：真实读者读完以后，能不能复述你的核心观点？这篇文章有没有冒一个具体判断的风险？关键事实有没有出处？结尾是不是把前面的环真正收住了？

学习的裁判可能是：不看答案，你能不能解一道新题？你能不能把这个概念讲给别人听？一周后，在有压力、没有 AI 提示的时候，你还能不能用出来？

做产品的裁判可能是：东西能不能跑？用户能不能完成完整流程？测试是不是先失败、修完以后再通过？快乐路径断掉以后，系统能不能恢复？

过日子的裁判甚至更简单：

这个计划有没有改变你今天的行动？

这一条很重要。很多 AI 使用，最后停在漂亮意图上。计划很清楚，下一步很明显，摘要很优雅。

然后，什么都没有变。

因为裁判缺席了。

## 第零阶级会设计裁判

我一直很喜欢“第零阶级”这个说法。

第零阶级不是最会收藏 prompt、最先用上新模型的人。他们不会用 AI 把自己变成答案消费者。他们用 AI 强化的是答案之前那一层：注意力、品味、记忆、练习、执行和判断。

在生成变便宜的时代，第零阶级不会由“谁有最新模型”来定义。这个优势会衰减得很快。模型会扩散，界面会改进，昨天的魔法会变成今天的按钮。

真正拉开差距的，会是裁判设计能力。

一种人向 AI 要输出。

另一种人搭一个带裁判的循环。

前者得到更多文字。

后者得到一个系统：它能发现错误，保存尝试，比较结果，并且随着时间变好。

刚开始，第二种人看起来可能更慢。他们会写下成功标准，保留测试样本，定义隐私边界，想清楚什么证据会改变自己的看法，设置停止条件。他们让 AI 不只是生成，而是一起维护那个用来评判生成物的“法庭”。

这看起来不如一个漂亮 prompt 神奇。

但它更容易复利。

## 坏裁判比没有裁判更可怕

最后还有一个提醒。

不是所有裁判都是好裁判。

坏裁判会用机器速度制造虚假的信心。只测快乐路径的测试，会祝福脆弱软件。只奖励关键词匹配的基准，会训练出浅薄回答。只奖励愤怒的社交指标，会惩罚细腻判断。只衡量忙碌程度的个人清单，会让你在最不重要的方向上显得很勤奋。

这本来就是所有测量系统的老问题，只是 AI 把它放大了。

人类优化坏指标，伤害会被人的速度限制。

agent 优化坏指标，伤害会被系统放大。

所以设计裁判时，必须带着一点谦卑。裁判不是神，它只是工具。它需要被检查，需要背后有保留的现实样本，需要被一些让它不舒服的例子挑战。它有时候可以被人类判断推翻，但不能随便推翻，更不能因为结果让你不方便就推翻。

最好的裁判，不是最苛刻的裁判。

最好的裁判，是最难让你假装自己在进步的裁判。

## 留在工程师的位置上

Loop Engineering 不是让你从工作里消失的技巧。

它是让你的工作上移一层。

你不再输入每一条指令，而是设计循环。

你不再凭感觉判断每一个草稿，而是设计裁判。

你不再因为 agent 说得流畅就相信它，而是给它一个可以公开出错、修正、再试一次的平面。

这才是 AI 让人变强的方式。

不是消灭阻力。

而是把阻力放到正确的位置。

循环给你运动。

裁判给你方向。

而人还有一个不该外包的工作：决定什么样的方向值得拥有。

所以，如果一件事会重复，就可以设计循环。如果下一步足够清楚，就可以让机器跑起来。

但在你走开之前，请留下一个裁判。

它要严格到足以让漂亮答案难堪，也要公平到愿意奖励真正的进步。

循环不会让系统进化。

带着裁判的循环，才会。
