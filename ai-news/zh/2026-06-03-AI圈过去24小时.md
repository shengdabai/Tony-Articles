# AI 圈过去 24 小时 · 2026-06-03

> 发布日期:2026-06-03 · 类型:AI 热点日报

---

今天是微软 Build 2026 大会日,各家发布会扎堆。但真正值得盯的主线,不是哪家又喊了句口号,而是 **agent 集体往「会自己组织流程、会随用变强」这个方向跨了一大步**:Claude Code 把多智能体编排做成了运行时可以即兴生成的「工作流」,NVIDIA、阿里把「自进化」直接写进 agent,独立开发者用一个 skill 替掉了上百块的付费软件。同期 Claude 同比增长 640%。一句话:**模型已经不是主角了——能编排、能记忆、能自我迭代的「系统」才是;而 Skills,正成为一个人撬动这套系统的杠杆。**

**1. Claude Code 上线「动态工作流」:让 AI 在运行时自己造一套多智能体框架**
Claude 官方博客发布新功能:Claude Code 现在能在运行时即兴创建并协调多智能体框架——通过执行一个 JavaScript 文件,生成一批各自拥有独立上下文窗口的子代理,专门对付单一上下文里跑久了会「智力惰性」的长任务,适合研究、安全分析、代码审查这类高价值复杂场景。为什么该关心:这是「工程化破界」最新的一记重拳——以前是你写死流程让 AI 执行,现在是 AI 按任务现场长出流程。**编排能力从「人预设」变成「机器即兴」,意味着一个人能驾驭的复杂度又上了一个台阶。** 对每天泡在 Claude Code 里的你,这条是今天的头号信号。 [source](https://claude.com/blog/a-harness-for-every-task-dynamic-workflows-in-claude-code)

**2. 「智能体工程」实战 22 条全录:一个高中后没发过软件的人,三个月做出 27K star 项目**
邵猛转述 @mvanhorn 的分享:核心是从「人主导编码」转向「人主导方向、智能体执行」,工作中心从 IDE 挪到终端 + 计划文件,方法论遵循 Research → Plan → Work 循环,靠一份 plan.md 约束智能体行为,配齐 22 条涵盖规划、并行、远程控制的实战技巧和完整工具栈。最炸的细节是:分享者三个月前还「高中毕业后没发布过有价值的软件」,如今做出了 27K star 的项目。为什么该关心:这是「用 AI 让人变强」最有说服力的实证——不是 AI 替你干,而是 AI 把一个普通人的产出上限直接抬高了一个数量级。**门槛塌了,但「想清楚方向」这件事的价值反而暴涨。** [source](https://x.com/shao__meng/status/2061974983094755575)

**3. NVIDIA 发布自进化 Hermes 智能体:越用越聪明的企业 AI**
NVIDIA Nemotron Labs 推出「自进化 Hermes 智能体」,主打一个理念——企业 AI 会随使用而持续改进,而不是出厂即定型。为什么该关心:自进化系统是值得长期死磕的方向。Agent 真正的分水岭,不在「这次答得多好」,而在「下次是不是比这次更好」。**一个会自我迭代的 agent,本质上是把「复利」装进了软件**——这正是「人和系统都该持续进化、复利成长」的机器版。(巧的是它也叫 Hermes。) [source](https://x.com/NVIDIAAI/status/2061870499232190967)

**4. OpenClaw 联手微软:Claws 现在能在企业内网安全跑起来了**
OpenClaw 宣布与微软合作,把自己带进微软和 Windows 生态——「你现在可以在公司内部运行 OpenClaw 了」,Claws 可以在企业环境里安全运行。为什么该关心:OpenClaw 是值得长期跟踪的 agent 演化样本。从「极客桌面玩具」走进「企业能合规部署」,是 agent 生态成熟的关键一跃。**一旦 agent 能进企业内网、被治理、被信任,它就从「试试看」变成了「基础设施」**——这条线值得长期记一笔。 [source](https://x.com/openclaw/status/2061869633624580452)

**5. 商汤开源 SenseNova-Skills:给任意 agent 装一套「AI 办公技能包」**
商汤开源了 AI 办公技能套件 SenseNova-Skills,专为 OpenClaw、HermesAgent 等「技能兼容」智能体设计,四大功能:信息图表生成(可镜像参考风格)、数据分析(多表解析清洗可视化)、PPT 创建(出大纲 + 智能排版,输出可编辑文件)、深度研究(跨学术技术社交多源搜索出报告),已完全开源。为什么该关心:这是 Skills 生态又一块拼图,而且是「跨框架通用」的——同一套技能,OpenClaw 能用、Hermes 也能用。**当能力被标准化成可插拔的 skill,「换底座不换能力」就成了现实**,这对不想被单一平台绑死的人是大利好。 [source](https://x.com/SenseTime_AI/status/2061822148076093625)

**6. 卡兹克:为了省那 120 刀,我把电脑清理软件做成了开源 skill**
中文独立开发者卡兹克受一条推文启发,用 Codex 对自己的 MacBook 做了只读存储分析,挖出 B 站缓存等一堆可清理空间(激进方案超 140G),随后做了个跨平台(Mac/Windows)的 AI 清理 skill 开源出来,替代收费的 CleanMyMac:扫描后生成可交互 HTML 报告,绿灯放心清、黄灯需判断、红灯禁止动,还带安全执行按钮。实测释放近 120G,而 CleanMyMac 只扫出 15.8G。为什么该关心:又一个「一个人 + AI = 一个小作坊」的鲜活样本,而且痛点极具体——把一次自己的真实需求,顺手做成了所有人能用的开源工具。**这就是 MicroSaaS 的种子形态:先解决自己的痒,再让它长成别人的药。** [source](https://x.com/Khazix0918/status/2061669881725309048)

**7. Claude Platform 出了 CLI:每个 API 端点都能从终端直接跑**
Claude Devs 宣布为 Claude Platform 推出官方 CLI(ant),让每个 API 端点都能从终端调用——直接调 Messages API、启动 Claude 托管智能体、把结果管道传进你的 shell;而且这个 CLI 被使用 claude-api 技能的编码智能体(也就是 Claude Code 本身)很好地理解。为什么该关心:这是 Claude API/SDK 生态又一次「terminal-native」化。对把整套工作流跑在 Claude 上的人,**API 能力越是「随手可调、可被 agent 理解」,自己搭工具的边际成本就越低**——它在悄悄把「用 Claude 搭东西」变成像写 shell 命令一样自然的事。 [source](https://x.com/ClaudeDevs/status/2061877343078244459)

**8. 数字摆出来了:ChatGPT 月活破 10 亿,Claude 同比增长约 640%**
据 Sensor Tower 估算,ChatGPT 2025 年 5 月全球月活破 10 亿,是史上达成该里程碑最快的应用;同期 Anthropic 的 Claude 月活达 5600 万,同比增幅约 640%,且已有用户在两者间来回切换。Anthropic 已秘密递交 IPO 申请,OpenAI 也在筹备上市。为什么该关心:抛开冰冷的数字,真正的信号是「脚下这块地」的位置——Claude 用户基数还只是 ChatGPT 的零头,但 640% 的斜率说明它正从「小众极客工具」往「主流」爬。**对深度押注 Claude 生态的人,这个增速既是底气,也是提醒:你早就站在了一条还在陡峭上升的曲线上。** [source](https://www.ithome.com/0/959/083.htm)

---

**今日 takeaway:** 把这八条串起来,讲的是同一件事——AI 的重心正在从「更强的模型」滑向「会编排、会记忆、会自进化的系统」,而 Skills 是普通人接入这套系统最便宜的接口。一个值得琢磨的问题:当 agent 开始能自己造流程、自己变聪明,**人剩下的、也最值钱的那部分能力,到底是什么?** 想清楚这个,你才知道该把省下来的精力,押到哪儿去。
