# AI Daily · 2026-06-26

> 发布日期:2026-06-26 · 类型:AI 热点日报

---

The strongest signal from the past 24 hours is that AI is moving further from “smart model” toward “operating system for real work.” Hooks, mobile control, agentic marketing, long-term memory benchmarks, and open coding models all point in the same direction: the real edge is not only better answers, but tighter loops between context, permissions, tools, verification, and feedback.

**1. Claude Code hooks turn AI into an event-driven work system**
Claude Code hooks are being used for permission alerts, startup briefings, context-compression summaries, folder cleanup, break reminders, and task completion notifications. Why it matters: this is not just a bag of automation tricks. It turns AI from a passive chat box into an event-driven layer around daily work. The useful lesson is to capture repeated actions as stable triggers, so AI becomes part of the workflow instead of another window to remember.
[source](https://mp.weixin.qq.com/s/LVj2foSXi_hBRKxjuYaUyw)

**2. Claude Code v2.1.193 improves auto mode, telemetry, and MCP reliability**
The new Claude Code release adds `autoMode.classifyAllShell`, expands rejection logging, introduces an optional OpenTelemetry event for assistant responses, improves real-time path completion, shows clearer MCP authentication prompts, and fixes several background task issues. Why it matters: once an AI coding tool becomes part of a real workstation, the hard question is not simply whether it can act automatically. The system must classify actions before execution, expose enough telemetry during execution, and leave traces that make failures debuggable afterward.
[source](https://github.com/anthropics/claude-code/releases/tag/v2.1.193)

**3. Codex is now generally available inside the ChatGPT mobile app**
Codex is now available in the ChatGPT mobile app, with device pairing, notifications, goals, side chats, file previews, and inline review comments. Why it matters: AI coding is no longer limited to sitting in front of a development machine. A phone can become the review, approval, and steering surface while the actual work continues on a laptop, desktop, or remote machine. That is a cross-time workflow: lightweight human decisions on mobile, long-running execution on the workstation.
[source](https://x.com/OpenAIDevs/status/2070254532911882707)

**4. Runway Agent 2.0 turns marketing creation into a test loop**
Runway Agent 2.0 helps create campaign concepts, generate ad variants, localize assets, produce social content, and propose the next round of creative tests. Why it matters: agents are entering the full loop of idea, asset, launch, data, and iteration, not just producing one image or one video. For readers, the practical shift is to treat content production as an experiment system. AI should not only make assets faster; it should shorten the feedback cycle that tells you what to make next.
[source](https://runwayml.com/news/introducing-agent-2)

**5. Ornith-1.0 targets open-source agentic coding**
Ornith-1.0 is an open model family focused on agentic coding, spanning multiple parameter scales and reporting strong results on SWE-Bench, Terminal-Bench, NL2Repo, and related coding-agent benchmarks. Its training approach jointly optimizes task scaffolding and final solutions with reinforcement learning. Why it matters: open coding models are moving from code completion toward building scaffolds, using terminals, and advancing tasks. That lowers the barrier for private deployment and gives small teams a more controllable path to AI programming assistants.
[source](https://huggingface.co/deepreinforce-ai/Ornith-1.0-35B-GGUF/blob/main/README.md)

**6. LongCat VitaBench 2.0 stresses long-term, dynamic agent memory**
VitaBench 2.0 evaluates agents with simulated users, complex tasks, changing preferences, executable tools, and long interaction histories. Why it matters: long-term memory is not just storing chat logs in a larger context window. A self-evolving system must deal with changing preferences, multi-day tasks, conflicting signals, and moments where the right move is to ask a question. Benchmarks like this push agents beyond short demos and toward long-term companionship in real workflows.
[source](https://mp.weixin.qq.com/s/HoiUxYnyJuh2_xdxmg8s8Q)

**7. Training general agents on game action data is gaining momentum**
One AI company is using massive game-action datasets, including precise input labels, to train agents that can operate in virtual environments and transfer parts of that learning to physical robots. Why it matters: game data is not valuable only because it is entertaining. It is dense behavioral data: goals, actions, feedback, timing, and changing environments. The deeper pattern is that AI systems may learn more stable action strategies by studying how people act under pressure, not only by reading text.
[source](https://techcrunch.com/2026/06/25/from-fortnite-to-robots-general-intuitions-2-3b-bet-that-video-games-can-train-ai-agents-for-the-real-world)

**8. An AI economy report shows fast revenue formation and sharper bottlenecks**
A new AI economy report argues that AI spending and revenue formation are accelerating quickly, while infrastructure depreciation, electricity, and data center costs remain major constraints. Why it matters: the point is not to stare at headline numbers. It is to understand where opportunity moves as model capability gets cheaper. Scarcity may shift toward trusted workflows, distribution, proprietary data, reliable delivery, and clear return on investment.
[source](https://x.com/rohanpaul_ai/status/2070288396644491317)

**Today's takeaway:** The AI battlefield is shifting from “which model is smarter” to “which system can close the loop.” Do you need a stronger model first, or a workflow that can execute, verify, remember, and improve?
